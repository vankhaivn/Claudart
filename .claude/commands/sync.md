Sync CLAUDART configuration from another AI layer into Claude Code.

Usage:

```text
/sync codex
```

The argument is the source side. `/sync codex` means: read the current Codex snapshot and update `.claude/`.

This command is intended for downstream user projects after they have copied CLAUDART into their repository. In the CLAUDART base template itself, maintain `.claude` and `.codex/.agents` manually as peer source templates instead of using sync as an authoring shortcut.

## Hard Rules

1. Read `.claudart/sync-map.md` first and follow it as the source of truth.
2. Do not use `git diff`, commit history, or session history to infer changes. Sync from the current filesystem snapshot only.
3. Do not copy, convert, or duplicate `.claudart/CONTEXT.md` or `.claudart/JOURNAL.md`.
4. Never create `.claude/CONTEXT.md`, `.claude/JOURNAL.md`, `.codex/CONTEXT.md`, or `.codex/JOURNAL.md`.
5. Never import `@.claudart/JOURNAL.md` into `.claude/CLAUDE.md`, `AGENTS.md`, `.codex/CODEX.md`, `.claude/rules/`, or `.codex/guidelines/`.
6. Source wins only for generated target files. If a target file lacks `Generated by CLAUDART sync`, report it and ask before overwriting.

## Procedure

### 1. Validate Invocation

- Accept only `codex` as the source when this command is running inside Claude Code.
- If the user asks for `/sync claude` from Claude Code, explain that Claude is already the active side and suggest running `$sync claude` from Codex instead.

### 2. Read Source Snapshot

Read these source files if present:

- `AGENTS.md`
- `.codex/CODEX.md`
- `.codex/guidelines/*.md`
- `.codex/commands/*.md`
- `.codex/agents/*.toml`
- `.agents/skills/*/SKILL.md`

Also read `.claudart/CONTEXT.md` only to confirm shared state exists. Do not read `.claudart/JOURNAL.md` unless the user explicitly asks for history.

### 3. Convert To Claude Native Files

Update or create:

- `.claude/CLAUDE.md`
- `.claude/rules/*.md`
- `.claude/commands/*.md`
- `.claude/agents/*.md`

Every generated target file must include a marker comment near the top:

```markdown
<!-- Generated by CLAUDART sync from <source-path>. Edit source side or run the opposite sync intentionally. -->
```

### 4. Preserve Claude Requirements

- `.claude/CLAUDE.md` must reference `@.claudart/CONTEXT.md`.
- `.claude/CLAUDE.md` must not reference `@.claudart/JOURNAL.md`.
- Rule files must have YAML frontmatter with `paths:` and `description:`.
- Agent files must have YAML frontmatter with `name`, `description`, `tools`, and `model`.

### 5. Verify

Run text-level checks:

- Search for stale shared-memory paths outside sync policy/command text: `.claude/CONTEXT.md`, `.claude/JOURNAL.md`, `.codex/CONTEXT.md`, `.codex/JOURNAL.md`.
- Search for operational illegal imports outside guardrail text: `@.claudart/JOURNAL.md`.
- Confirm `.claudart/CONTEXT.md` and `.claudart/JOURNAL.md` exist.
- Confirm `.claudart/CONTEXT.md` is at most 150 lines.

### 6. Report

Summarize:

1. Source side read.
2. Files created or updated.
3. Manual target files skipped.
4. Orphan candidates found.
5. Verification result.
